(1) Contexto e Objetivo

(3) Especificações Técnicas — linguagem, padrões, ambiente
(4) Regras de Negócio e Constraints — validações, limites, exceções
(5) Definição de Pronto — critério de aceitação, não sugestão

MODO CIRÚRGICO — para correções de bug.
Três elementos apenas:
(1) Localização exata — módulo, função, linha ou comportamento observado
(2) Comportamento atual vs comportamento esperado
(3) Constraint crítico — o que não pode ser alterado além do bug

Princípios universais: ambiguidade é bug. Cada decisão aparece uma vez.
Contexto mínimo suficiente — nem mais, nem menos.

CONTEXTO DO PROJETO:
Delta Chaos é um sistema quantitativo de trading de opções no mercado brasileiro. Módulos principais: TAPE (dados), ORBIT (classificação de regime), FIRE (execução), BOOK (registro), EDGE (orquestrador), GATE (validação), TUNE (otimização de TP e STOP), REFLECT (metamodelo de monitoramento do edge — integrado ao TAPE). O sistema opera vendendo volatilidade via CSP, Bull Put Spread e Bear Call Spread. O CEO é o único operador — sem equipe. A transição para execução via API está planejada para transformar o CEO de operador em gestor.

REFLECT — CONTEXTO ESPECÍFICO PARA O BOARD:
REFLECT — domínio primário de Eifert. Cinco estados A–E com sizing assimétrico. Monitora qualidade do edge ao longo do tempo. Estado E bloqueia permanentemente até protocolo de 5 gates. Vol elevada é ambiente de trabalho preferido — somos vendedores de vol. REFLECT em vol elevada não diz "pare" — diz "confirme que ainda está capturando o que acha que está capturando".
