# Plano: Mover Orquestrador para dentro de ATIVO

## Problema Atual

O botão "Orquestrador" está como uma aba interna separada (junto com "Visão Geral" e "Ativo") em `MainScreen.jsx`. Ele usa `activeTicker` que por padrão é "VALE3", ficando preso a esse ticker.

## Objetivo

Mover o Orquestrador para a navbar inline do `AtivoView.jsx`, entre "ANALYTICS" e "MANUTENÇÃO", de forma que os processos (GATE, TUNE, ORBIT, EOD) sejam executados para o ticker selecionado no dropdown, assim como as demais abas (ORBIT, REFLECT, CICLOS, ANALYTICS, MANUTENÇÃO) são atualizadas conforme o dropdown.

## Mudanças

### 1. `atlas_ui/src/layouts/MainScreen.jsx`

- Remover `{ id: "orquestrador", label: "Orquestrador" }` do array `internalTabs` (linha ~217)
- Remover o bloco de renderização `{internalTab === "orquestrador" && <OrquestradorView ... />}` (linhas 351-353)
- Remover o componente `OrquestradorView` inteiro (linhas 32-92) — será recriado dentro de `AtivoView.jsx`

### 2. `atlas_ui/src/components/AtivoView.jsx`

- Adicionar `"orquestrador"` ao array de tabs inline, entre `"analytics"` e `"manutencao"` (linha 1108):
  ```js
  ["orbit", "reflect", "ciclos", "analytics", "orquestrador", "manutencao"]
  ```

- Criar o componente `OrquestradorTab` (antes do `AtivoView` principal) com:
  - Props: `{ ticker }` (o ticker selecionado no dropdown)
  - Botões: RUN GATE, RUN TUNE, RUN ORBIT, RUN EOD — todos usando o `ticker` prop
  - Exibir o nome do ticker nos botões: `▶ RUN GATE (TICKER)`
  - Incluir `<LogPanel />` abaixo dos botões para streaming de logs
  - Usar o mesmo `btnStyle` e lógica de `runSubprocess` do `OrquestradorView` original
  - Estado `loading` para desabilitar botões durante execução

- Adicionar a renderização da nova tab no bloco de conteúdo (após analytics, antes de manutencao):
  ```jsx
  {subTab === "orquestrador" && <OrquestradorTab ticker={activeTicker} />}
  ```

- Importar `LogPanel` no topo do arquivo:
  ```js
  import LogPanel from "../components/LogPanel";
  ```

## Resultado Esperado

A navbar inline do ATIVO ficará:
```
ATIVO: [dropdown] | ORBIT | REFLECT | CICLOS | ANALYTICS | ORQUESTRADOR | MANUTENÇÃO
```

Quando o usuário selecionar um ticker no dropdown e clicar em ORQUESTRADOR, os botões mostrarão o ticker selecionado (ex: "▶ RUN GATE (PETR4)") e executarão os processos para aquele ticker específico.
